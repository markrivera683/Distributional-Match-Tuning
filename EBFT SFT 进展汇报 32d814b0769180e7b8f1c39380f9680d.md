# EBFT / SFT 进展汇报

## 1. 背景与目标

这阶段工作的核心目标，是在 AoPS 数学问答场景下，探索一条从 paper-faithful EBFT baseline 出发、逐步引入更强分布建模能力的训练路线，并观察其是否能够转化为最终的数学答题能力提升。

这条线的出发点有两个：

1. 原始 EBFT / distributional reward 的思路，本质上不是只看单点答案是否正确，而是希望从“分布匹配”的角度去建模生成行为。
2. 我们进一步考虑到，在数学推理任务里，模型的输出并不总是一个单点对象，而更像是一簇可能的生成路径、局部 token 轨迹和中间表征分布。因此，单纯的 pointwise reward 可能不足以表达我们真正希望模型靠近的目标分布。

在这个意义上，我们的方法设计受到 **distribution matching / distribution-level target** 思想的启发，而这部分更直接的灵感来源，是师兄关于 **NCFM** 的工作。我们不是简单把 reward 看成一个标量打分问题，而是尝试把训练目标理解为：**让模型生成分布在某种特征空间或目标测度意义下，更接近我们想要的分布。**

构建了 G1 / G2 / G3 三个逐步递进的版本。

## 2. 方法设计：G1 / G2 / G3

我们这条方法线的出发点，是把 EBFT 从“点目标对齐”逐步推进到“分布目标对齐”。

原始 EBFT 已经不再是单纯的 token-level CE，而是通过 critic 提取的特征来定义训练信号。但如果进一步从第一性原理看，数学推理任务中真正值得对齐的对象并不一定是某一个单点答案，而更像是一个条件输出分布。也就是说，我们更关心的不是“某个样本和某个目标点有多近”，而是“模型生成分布是否更接近我们想要的目标分布”。

### G1：Paper-faithful baseline

G1 的定位是一个尽量干净、稳定、接近原始 paper regime 的参考基线。

它并不追求方法创新，而是先回答一个更基础的问题：在一个尽量保守、可控的设置下，EBFT 本身是否能稳定工作。整体训练过程尽量贴近原始 EBFT 的工作区间

### G2：把 pointwise reward 改成 distribution-level discrepancy

G2 的核心改动是把训练信号从“单样本对单目标的对齐”改成“生成样本集与目标分布之间的差异”。

它的核心思想是：

**不再直接奖励“单个样本是否接近某个 target point”，而是比较整组生成样本在特征空间中的经验分布，与目标分布之间的 discrepancy，把 EBFT 的 reward 从 pointwise alignment 推进到 distributional matching。**

具体来说，G2 使用 characteristic-function 风格的分布差异来定义奖励。

这一差异不是作用在单个样本上，而是先作用在一组生成样本与目标集合之间，然后再通过 leave-one-out 的边际贡献方式，把 group-level discrepancy 转换成每个样本的 reward。

这一步有两个重要效果：

1. **多样性不再是外加项，而是由分布差异本身内生决定。**
    
    如果某个样本能够帮助整组生成结果更好地覆盖目标分布，它就会得到正向奖励；反之则会得到较低甚至负向奖励。
    
2. **训练对象从单点 target 转向 target measure。**
    
    这意味着模型优化的不再是“往某一个答案特征靠拢”，而是“让整体生成分布在特征空间里更接近目标分布”。
    

### G3：在 distributional reward 基础上进一步学习 feature geometry

G2 解决了一个问题：reward 不再是 pointwise。

但它仍然默认 critic 提供的 feature geometry 是合适的。

这其实是一个很强的假设，因为如果 feature space 本身不能很好表达目标分布与生成分布的差异，那么即使 reward 公式已经 distributional 化，训练信号也可能仍然受限。

G3 的核心判断是：

**如果目标是做 feature-space distribution matching，那么 feature geometry 本身也应该允许被轻量地调整。**

但这一步不能走得太激进，如果直接把 critic 全量放开训练，系统会立刻变得更不稳定，也更难判断收益到底来自 reward 设计还是来自 critic 漂移。

因此，G3 采用的是一种非常克制的做法：

- critic backbone 尽量保持稳定
- 只在 critic feature stream 上增加一个轻量的 residual bottleneck adapter
- 只训练小规模的 head / adapter，而不是大幅放开整个 critic
- 同时保留一个较稳定的 EMA target geometry，用作较慢变化的参照

G3 在 critic 的 feature stream 上加入一个小型 residual bottleneck adapter。

这个模块的设计目标不是重新学习整个表示空间，而是做一种**局部的几何校正**。

它的好处在于：

- 参数量小
- 初始化时接近恒等映射，不会一开始就破坏原 feature geometry
- 训练中可以逐步学出一个更适合 distribution matching 的特征变换

G3 并没有选择把 critic backbone 全量解冻，相反，它采用的是一种更保守的结构：

- backbone 尽量冻结或近似冻结
- 主要训练 classifier head 和 feature adapter
- 减少训练不稳定性
- 减少表示空间整体漂移
- 提高 distributional reward 对 feature geometry 的适配能力

### 3. 用 EMA target geometry 稳定 discrepancy learning

G3 引入了 EMA target branch，用一个较慢变化的目标几何作为参照。

- online 分支负责更新
- EMA 分支提供较稳定的 target geometry
- discrepancy learning 不会因为 target 本身过快漂移而失稳

### G123对比

- G1 是一个点目标对齐基线
- G2 是 reward 层面的 distributional upgrade
- G3 是在 G2 基础上进一步做 representation / geometry 层面的 distributional upgrade

## 3. 实验设置

### 3.1 实验目标

本轮实验的核心目标，是比较以下几类方法在 AoPS 数学问答场景下，是否能够提升最终 actor-only 的答题准确率：

### 3.2 数据与模型

训练侧采用统一的数据与基座模型体系，保证 G1 / G2 / G3 之间的可比性。

评测侧统一使用 AoPS-Instruct-test，并采用同一 actor-only benchmark 协议。

统一使用Qwen 3.5 2b base 模型。

### 3.3 训练预算

**G1,2,3均控制训练global step为500**：

- 每组实验的训练硬件均为 **8 × A100**
- 每组训练时长都 **超过 7 小时（8卡并行）**

### 3.4 评测设置

评测时统一采用 actor-only final-answer accuracy benchmark：

- 只评测 actor
- 不使用 critic 参与最终打分
- 使用统一测试集
- 统一 prompt / generation / scoring 协议

后续我们还对评测协议进行了修正，以排除评测实现本身造成系统性偏差的可能性。

## 4. 当前结果与解释

在统一 actor-only AoPS benchmark 下，得到：

| 模型 | 总样本 | 正确数 | 准确率 |
| --- | --- | --- | --- |
| Base | 5328 | 447 | 8.39% |
| G1 | 5328 | 448 | 8.41% |
| G2 | 5328 | 427 | 8.02% |
| G3 | 5328 | 430 | 8.07% |
| SFT | 5328 | 240 | 4.50% |

实验结果非常奇怪 G2，G3完全无提升 这个可以接受 因为还没有调整工程细节如参数等 但是G1（EBFT原方法）几乎也没有任何提升 sft更是大幅下降 而且所有微调后的模型均差于Base模型 而且所有实验结果都非常差，

当前结果显示，不仅 G1/G2/G3 没有显著超过 Base，连 Base 本身在 AoPS 上的绝对准确率也很低，这说明问题可能是**任务难度、模型能力与训练目标三者之间存在错配**。从原始 EBFT 论文看，它验证的方法主要是在 **Q&A coding、unstructured coding 和 translation** 任务上，而不是 Olympiad-level math 这类高难度数学推理任务。（他们用的是qwen 2.5 1.5B模型 所以我们这里类似的选用了2b的qwen3.5 但是效果很差）

因此，我们现在把一个 **Qwen3.5-2B Base** 模型直接放到 AoPS 这种高难、长推理、强符号约束的场景里，本身就可能已经超出原方法最初验证的工作区间。另一方面，AoPS 数据集论文本身也并不是在“弱 base 小模型”上得到结论的：他们用于 SFT 的是 **DeepSeek-Math-7B-Instruct、Mathstral-7B，以及 Llama-3.2 1B/3B-Instruct** 等指令模型/数学专门模型，并且还使用 **Qwen2.5-72B** 对解答进行重写，且常常将 **AoPS-Instruct 与 Numina** 混合训练，训练预算约为 **3 个 epoch（约 18,000 steps）**。因此，当前结果更像是在提示：**AoPS 任务本身比原始 EBFT 论文任务更难，而我们当前使用的 2B base 模型也明显弱于 AoPS 论文里采用的 math-specialized / instruct 模型；在这种前提下，微调没有打出增益并不完全意外，真正值得进一步确认的是：问题究竟来自任务过难、模型过弱，还是当前训练目标确实没有有效转化为最终数学答题能力。**
